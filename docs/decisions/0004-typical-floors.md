# ADR-0004: 基準階は一度だけ書く — レベル範囲・スパン展開・stack

日付: 2026-07-23 / 状態: 採用 (v0.2)

## 文脈

10階建ての集合住宅を書くと、反復が初めて現れる。2〜9Fの基準階は同じ割付であり、8回複製すれば書けるが、それは設計基準「LLMが一つのコンテキストで読み切り、正しく部分編集できるか」を壊す。基準階の一つの変更が8箇所の編集になり、diffは8倍に膨れ、「この住戸が広がった」という一言で読めなくなる。原本に複製を持たないことは、テキストであることと同じくらい本質的だった。

## 決定

三つの機構を足す。**レベルの等差宣言** `level L3..L9 6700 pitch:2900 h:2400 slab:500` は7つのレベルを一度に宣言する。**パスのスパン展開** `/L2..L9/A` は、宣言済みレベルのz順の並びとして解決され、その行が各レベルへ展開される — 一行の中の複数パスは同じスパンでなければならず、同じ階に揃って展開される (`boundary /L2..L9/A /L2..L9/corridor` はA|廊下を8フロアぶん張る)。字下げの開口・area・segも展開先の全てに付く。**stack宣言** `stack ev L1..L10 type:shaft` は垂直に積層する空間列を宣言し、連続するレベル対に垂直境界を張る。

DSLは「基準階は一つ」という意図を保持し、機械形式 (正準JSON) は展開後を持つ。原本のdiffは基準階の一行の変更として読める — 159行の原本が90空間・244境界に展開される集合住宅で、界壁の仕様変更は1行のdiffである。

## 代替案

8回書く案は上記の通り編集の局所性を壊す。composition (レイヤー参照) で基準階レイヤーを8回重ねる案は、OpenUSD/IFCXの流儀に忠実で機械形式側の将来実装とも整合するが、DSLの一読性を今失う理由がない。スパン展開はcompositionの前借りであり、正準JSON側で参照レイヤーに正規化する道は塞いでいない。

## 帰結と代償

**例外階がまだ書けない。** 8Fだけ間取りが違う、5Fに吹抜けが抜ける — スパンから特定階を除く記法 (except) か、展開後への上書きが要る。これは吹抜け (`type:void` — 基準階の床が一部抜ける) と同根の問題であり、同時に設計する。スパンはパスのレベル段 (先頭セグメント) にしか使えず、連番のみ。stackの端点レベルは宣言済みである必要がある (levelを先に書く)。
