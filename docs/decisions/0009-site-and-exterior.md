# ADR-0009: 敷地と外部の分節 — 外もまた空間である

日付: 2026-07-23 / 状態: 採用 (v0.5) / 対応Issue: MUN-149, MUN-150

## 文脈

/out は一枚岩の外部で、どの外部に面するかが言えなかった。延焼ライン (隣地側だけ仕様が変わる)、接道、屋外階段の「地上」の意味、いずれもこの分節を待っていた。IFCではIfcSite・IfcExternalSpatialElementにあたるが、敷地境界線そのものはIFCにも一次エンティティが無い — 空間一次が素直に勝てる場所である。

## 決定

**1. /out は複数のexterior空間に割れる。** `space /out/road exterior name:南側道路 road:6000` — 方角・性格ごとに割り、粒度は自由 (一枚岩のままでも良い — 既存例は無修正で有効)。道路は `road:幅員` の解釈属性で印す。上階の外壁は方角別のexteriorに `edge:` で面する。

**2. 地上の外部空間 (庭・通路) はL1上の実在の空間として建物の周りをタイルする。** 領域を持つので、建物との重なりはエラーとして検証され、室との境界線分は矩形同士から正確に導出される。GL/FLの段差はv0では無視 (測地・高さ基準はMUN-159で再訪)。「地上」の意味はこの空間たちが担う — 屋外階段や玄関はここへ開き、避難は庭を通って門扉から道路へ抜ける (「寝室から道路まで扉3枚」)。

**3. 敷地 = site:1 属性を持つゾーン。** `zone /site name:敷地 site:1 area:126.24` — 幾何を持たない集約という原則を保ちつつ、宣言面積 (測量値) を属性に持ち、`site` コマンドが導出面積 (敷地内空間+建物投影の合併) と突き合わせる。高さ不変量と同じ「宣言+検査」のパターン。

**4. 敷地の数字は導出する。** `site` コマンドが敷地面積 (宣言/導出の照合)・接道 (敷地空間と道路の境界線分長 — 建物外壁が道路に面する分は数えない)・建築面積 (屋内空間の水平投影の合併)・建蔽率・容積率を返す。基本計画のボリューム検討の数字が、構成を書けば出てくる。

**5. 塀・門扉は境界の語彙で書く。** 塀 = `t:120 spec:ブロック塀 air:1 h:1200` (物はspec、外気は遮らない)、門扉 = その境界の door。外構 (MUN-151) の語彙の入口。

**6. 庇下の導出。** 採光の半屋外係数0.7は「上に空間がある半屋外 (バルコニー下・庇下)」にだけ適用し、上が開いた半屋外 (庭・最上階バルコニー) は1.0とする。屋根の有無すら宣言ではなく、上階の空間の重なりから導出する。

## 代替案

敷地を専用の第4要素 (site) として幾何ごと導入する案は、空間・境界・ゾーンの三つ組で足りるのに要素を増やす。外部をレベルごとに割る案 (地上/2F上空/…) は無限の分節を招く — 地上性は敷地空間が、方角性は/outの分節が、それぞれ一度だけ担えば足りる。

## 帰結と代償

敷地空間のタイルは規律を要する (宣言/導出の照合が破れを警告する)。建築面積の算入 (庇・バルコニーの1m/2m規則) は粗いまま。延焼ラインの実計算は距離幾何が要るため未実装 — ただし「どの隣地に面するか」はこの分節で言えるようになった (足がかり完了)。傾斜地・GL段差は未対応。集合住宅の例の/out分節への移行は必要になった時に行う (一枚岩の/outも引き続き有効)。
