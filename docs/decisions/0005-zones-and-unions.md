# ADR-0005: 間取りの粒度 — ゾーンとL字の合併

日付: 2026-07-23 / 状態: 採用 (v0.3)

## 文脈

住戸の中の間取りを書くと、粒度の二層性が本格化する。住戸は面積計算・分譲・管理の単位であり続けたいが、間取りを書けば室が一次要素になる。両方が同時に「数える単位」でありたい。あわせて、L字のLDKのような非矩形の室が初めて必要になった。

## 決定

**1. ゾーン = 数える集約。** `zone /L2..L9/A name:Aタイプ use:exclusive` は幾何を持たない宣言で、パス接頭辞で室を束ねる。ゾーンの面積は配下の室の合計として導出され (壁芯で室が住戸を敷き詰めるので住戸壁芯面積に一致する)、`use` などの属性は配下の室へ継承される (室自身の宣言が勝つ)。空間の階層パスがそのまま集計の階層になる — 「共用と専有の面積比」は住戸を間取りに割った後も1行も変えずに答えられる。

**2. 粒度は混在してよい。** 集合住宅の例ではAタイプだけを間取りまで書き、B〜Eは住戸一室のまま。基本計画の進行に合わせて、詳細化した住戸から順に割っていける。ゾーンがあるから、割っても集計の言葉 (Aタイプ 34.8㎡) は壊れない。

**3. L字は矩形の合併。** 領域は `+` で複数書ける: `space .../ldk ldk X1+2600..X2 Y1..Y2-1800 + X1..X1+2600 Y1..Y1+1400`。面積は合計、壁は合併の外周と共有辺から導出され、共線の外周は一本にマージされる (掃き出し窓が2矩形をまたぐS面に普通に置ける)。同一ペアがL字で2辺接する場合 (LDK|洋室)、扉は `edge:` で辺を選ぶ。

## 代替案

住戸を空間のままにして間取りをarea (数えない分節) で書く案は、室が通行と面積の単位である以上不可 (寝室の扉が書けない)。多角形を直接書く案は、直交グリッドの言語 (通り芯とオフセット) を捨てて頂点列に落ちるため退けた — 矩形の合併は骨格の言葉を保ったまま同じ形状クラスを覆う。

## 帰結と代償

矩形合併は直交形状しか作れない (これは方針: 曲線・斜めは対象外)。合併の矩形同士の重なりはエラーとし、面積の二重算入を防ぐ。ゾーンの入れ子 (棟>住戸>室) はパス上は自然に書けるが、集計の重複規則はまだ決めていない。
