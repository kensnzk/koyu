# ADR-0011: 敷地形状 — 所与のジオメトリの隔離レイヤー

日付: 2026-07-23 / 状態: 採用 (v0.7)

## 文脈

敷地はまず矩形にならない。ADR-0009の敷地は「矩形タイルの合併」で面積を導出していたが、斜めの隣地境界・不整形な角地では宣言面積 (測量値) と照合できず、建蔽率・容積率の分母が正しく出ない。一方で「形は生成物」はこの記法の背骨であり、多角形の記述を安易に入れれば建物の形まで書かれ始める。

## 決定

**1. 敷地形状は「所与」であり「設計」ではない — この区別が線を引く。** 建物の形は空間の構成から生成される。敷地の形は測量に由来する世界からの入力で、生成のしようがない。だから敷地多角形は、この記法で唯一「書かれる形」として認める: `polygon /site x,y x,y x,y ...` (mm座標、3頂点以上)。

**2. 隔離レイヤーに置く。** 敷地形状は別ファイル (site-geometry.muro) に書いて `import` する運用を標準とする (ADR-0010の合成の応用)。測量図から書き写す層が分離されていれば、設計の層 (空間と境界) にジオメトリが混ざらない。重複はコンフリクト (出所つきエラー)。

**3. polygonはゾーンに付く。** 幾何を持たないというゾーンの原則 (ADR-0005) の例外はつくらない — polygonはゾーンの属性的な付帯物で、`site:1` のゾーンに対応させる。対応するゾーンが無ければ警告。

**4. 面積の真は多角形が持ち、タイルは近似に格下げされる。** `site` の導出面積はシューレース公式で多角形から出し、宣言面積 (area: 測量値) と照合する。地上の外部空間タイル (ADR-0009) は引き続きグラフ・接道・採光を担うが、面積の照合からは外れる — 斜め境界にタイルが届かなくても良い。

**5. 建物のはみ出しは検査する。** 領域を持つ空間のうち、敷地ゾーン配下でも外部でもないもの (= 建物とその張り出し) の全ての角が多角形の内側になければエラー (境界上は内側扱い、許容1mm)。位置が言葉になれば、はみ出しも言葉で指摘される — 明示位置 (ADR-0010) と同じ原理。

**6. 配置図に敷地境界線を引く。** 最下階の平面 (配置図兼用) に一点二点鎖線で多角形を描く。形は生成しない — 書かれた所与の形をそのまま引く。

## 代替案

**空間の領域を多角形に拡張する**案は、矩形の区間代数 (共有辺・外周・合併) の全面書き換えであり、M1.5で見送った「真の非直交」を裏口から呼び込む。**敷地を第4要素にする**案はADR-0009で棄却済み — ゾーン+属性で足りる。**タイルを多角形にクリップする**案は多角形クリッピングの導入で、得るのは僅かな描画の正確さだけ。

## 帰結と代償

不整形敷地の建蔽率・容積率が正しい分母で出る (ショーケース: 五角形1,097.80㎡の角地)。接道は引き続き矩形タイルと道路の境界線分から出る — 斜めの道路境界の接道長は近似のまま (代償)。建物の斜め配置 (グリッド非平行) は相変わらず書けない — これは敷地形状ではなく建物の問題で、M1.5の判断のまま。GL段差・測地座標は未対応 (MUN-159)。
