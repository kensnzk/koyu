# 2026-07-23 — 複数フロアと高さの一貫性

二つの仕事をした。ひとつは昨日の問いへの答えとして、同じ二室一扉をIFC4とIFCXで生成した三方比較 (examples/comparison/ — 0.6KB / 7.3KB / 20KB、IFCXの72%はメッシュ座標)。IFC4はThatOpenのweb-ifcでパース検証した。

もうひとつはM1の前半。2フロアの小さなオフィス (廊下+執務室+会議室+コア) を約100行で書いた (examples/office.ifcxs)。記法v0.1として、通り芯オフセット (`X1+3200`)、レベルの高さ宣言 (z/h/slab)、垂直境界 (`type:stair` / `type:shaft`、床は書かず既定) を足した。

核心は高さ方向の一貫性の担保 (ADR-0002)。BIMは3Dであることでこれを暗黙に解決していたが、空間一次のテキストでは「宣言された不変量: 天井高 + 上階slab ≤ 階高」の検査に置き換えた。食い込みはエラー、積み上がりは `levels` コマンド (テキストの矩計) で見える。EVシャフトは「連続するが通れない」ので、`doors /L1/ev /L2/ev` は階段迂回の4枚と正直に答える。テストは27本。

位置づけも定めた: このデータの解像度は基本計画レベル。垂れ壁を表現しないのは省略ではなく抽象度の選択で、計画初期にBIMが重すぎたという既存の弱点の裏側が主戦場。使い勝手の参照点はVectorworks的な軽さ。
