# 2026-07-22 — 最初の一手

原稿「建築を書く」を起点にプロジェクトを立ち上げた。

IFC5-development をクローンして hello-wall 一式と schema/ifcx.tsp を読み、借りる機構 (パス名前空間+レイヤー合成) と借りないもの (UUID主キー・メッシュ同梱・建築物オントロジー) を切り分けた (ifcx-notes.md)。そのうえで二室一扉を三つの形式で書き比べ、author形式=独自DSL・機械形式=正準JSONに決めた (spec/notation-v0.md, decisions/0001)。

最小ツールを実装した。パーサ、整合チェック (`ifcxs check`)、グラフ問合せ (`doors` `graph` `stats`)、平面SVG生成 (`plan`)、正準JSON (`json`)。テスト14本。壁を描く操作はどこにも無く、壁は二空間の割付から導出される。「/L1/a から /out へ扉2枚」が、変換なしにグラフへの問いとして答えられることを確認した。

二室一扉はコメント込み22行。生成した平面図は docs/img/two-rooms.svg にスナップショットを置いた。

開いたままの問い: 吹抜け・半屋外・粒度 (M1で正対)、L字領域、開き勝手、`/out` の分節。次の一手は一フロアを書くこと。IFC_samples/ にBLCJのRevitサンプルがあるのを確認 — M5のコーパス候補。

Linearに概要とマイルストーンM0〜M5を立てた。
