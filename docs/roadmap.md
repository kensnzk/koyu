# IFCXS ロードマップ

原稿 (writing-architecture.md) の「最初の一手と、その先」を行程に切ったもの。Linear の IFCXS プロジェクトのマイルストーンと対応する: https://linear.app/munipersonal/project/ifcxs-2789f588a03a/overview

これは探求であり、締切は設けない。各段の終わりにあるのは日付ではなく分水嶺の問いで、答えられたら次へ進む。

## M0 素振り — 二室一扉 (2026-07-22 ひと通り)

IFCXのサンプルを読み (ifcx-notes.md)、二つの室とその間の扉一つを自分の記法で書いた。空間の同一性・境界の帰属・接続・パスの切り方はここで決まり (decisions/0001)、記法の書き比べと決定は spec/notation-v0.md に残した。最小ツール (パーサ・整合チェック・グラフ問合せ・平面SVG生成・正準JSON) が動き、テスト14本とCI設定がある。分水嶺の問い「数十行で四つの決定が言葉になるか」には、なった、と答えられる。残: IFC5 alphaビューアでのレイヤー合成の実見。

## M1 一フロア〜複数フロア — 曖昧さへの正対 (2026-07-23 前半済)

一住戸〜一フロア以上を書き、IfcSpaceが実務で機能しにくかった当の曖昧さに定義を与える段。位置づけをここで明確にした: **このデータの解像度は基本計画レベルである。** 垂れ壁や建具詳細を表現しないのは省略ではなく抽象度の選択であり、計画初期にBIMが重すぎるという既存の弱点の裏側こそがこの記述の主戦場になる。使い勝手の参照点はVectorworks的な軽さ。

前半 (済): 2フロアのオフィス (廊下+複数室+コア) を約100行で記述。通り芯オフセット (`X1+3200`)、複数レベル、高さ方向の一貫性 — BIMが3Dであることで暗黙に担保していたもの — を「宣言された不変量 (天井高+上階slab ≤ 階高) と自動チェック」に置き換えた (ADR-0002)。床は書かず垂直隣接は導出、例外 (`type:stair` / `type:shaft`) だけ宣言。`levels` コマンドがテキストの矩計。避難の問い「2階執務室から外へ扉4枚 (階段経由、EVは通れない)」に即答できる。

後半 (残): 集合住宅の基準階 (住戸=ゾーンの二層粒度がここで必然になる)、吹抜けと階をまたぐ空間 (`type:void` の設計)、半屋外・外部空間の分節、L字型の室 (多角形/複数矩形)、開き勝手。分水嶺: これらを書き切っても記法が破綻しないか。

## M1.5 直交からの最初の逸脱 — 斜め

曲線は扱わない (原稿の「扱わないこと」) が、斜めの壁・雁行する外形は基本計画に現れる。多角形領域と斜め境界を、グリッドの言語を壊さずにどう書くかを設計する。M1後半のL字領域の自然な延長として置く。

## M2 一棟 — グラフへの問い

一棟を書き、グラフに問いを投げる。この室からこの室まで扉をいくつ通るか、防火区画をまたがずに外へ出られるか、共用と専有の面積比はいくつか。**部材のモデルなら抽出作業が要る問いに、何もせず答えられるかどうかが、オントロジーとしての分水嶺** (原稿)。避難・面積・区画・動線が同じ記述の異なる読み方になることを確かめる。

## M3 平面図 — 実務の言語への翻訳

素描 (M0のSVG) を図面に近づける。寸法線、通り芯記号、建具の作図表現、縮尺。平面図が生成できれば人に見せられ、実務の言語に翻訳できる。生成が一意でないこと (同じ構成から複数の形が出ること) をここで実際に確かめる。

## M4 版管理とレビュー

テキストであることの帰結を回収する。差分を「この室が広がった」という構成の言葉で表示するdiffツール、Pull Requestによる設計レビューの試行、コミットごとに整合と法則を確かめるCI (芽は .github/workflows/ci.yml と `ifcxs check` にある)。

## M5 IFC取り込み — コーパスの獲得

一方向・非可逆のIFC→IFCXS変換を書く。往復互換は最初から捨てる。BIM/IFC系のツールはThatOpenのOSS (web-ifc, @thatopen/components) を使う。コーパスの最初の候補は手元の IFC_samples/ にあるBLCJサンプル (RC造3000㎡・S造1000㎡)。IfcSpaceが書き出されていないモデルから空間をどう復元するかが、おそらく本当の難所。参照リポジトリをクローンする場合は ~/Documents/github に置く。

## 地平 (マイルストーンにしない遠景)

デジタルツイン (実測を設計の上に重ねる一枚のレイヤー)、都市 (数百行の軽さでCityGML・歩行空間ネットワーク・不動産IDと同じ層に載る)、フィジカルAI (空間グラフ+通行可能性+意味がそのまま行動の地図になる)。いずれも原稿の「地平」の節が原典。M2のグラフが答えられるようになったとき、最初の接続実験を選ぶ。

## 扱わないこと (再掲)

オーサリングツールは作らない。往復互換は捨てる。曲面と自由形状は対象外。設計意図の記録はデータではなくプロンプトの側。事業化は当面目的にしない。
